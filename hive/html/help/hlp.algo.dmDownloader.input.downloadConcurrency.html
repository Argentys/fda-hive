hlp.algo.dmDownloader.input.downloadConcurrency Download concurrency option Specify how many files should be loaded to the system at the same time This option allows the user to specify whether to load one or several files to the system at the same time. Low specifies to download jobs individually. High tells the downloader to do several jobs concurrently. PLEASE NOTE: if the user has several redundant items in the list, the high option will collapse the system. Furthermore, if the downloaded file is big, the user should not submit the same request before the previous request has been completed, otherwise it will collapse the system.
