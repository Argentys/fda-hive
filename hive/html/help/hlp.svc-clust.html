hlp.svc-clust.html HIVE-Octagon Sequence hierarchical clustering HIVE-Octagon allows the user to perform comparative analysis of multiple variant calling (profiling) results. The user has the ability to visualize the hierarchical relationship between SNP frequences in different samples in the form of a downloadable phylogram or NEWICK raw data file.

Parameters may be modified to customize hierarchical clustering of sequences of SNP frequencies for alignment profiling results.

    Noise filtering for unfiltered profiling data

    This option allows noise filtering to be applied to any profiling results that were generated without filtering, e.g. in order to compare a collection of profiling results among which some were generated with noise filtering and some were not.

    • Automatically filter 95% noise — data from those profiles that had not already had noise filtering done when they had been generated will pass through an automatic noise filtering stage.
    • Do not filter noise — no additional noise filtering will be done; this may lead to flawed comparison between profiles that already had filtering done when they had been generated and those that had not.

    References with incomplete profiling data

    This option controls what happens if some alignment profiling results have data against a reference and others do not.

    • Treat missing data as SNP-free — profiling results which do not have data for the reference will be treated as if the data existed and was identical to the reference genome, i.e. SNP-free.
    • Skip in calculation — alignment profiling results against a reference will be skipped unless all of the selected profiling results have data against that reference.

    Minimum coverage

    SNP data for positions with coverage below the specified minimum will be ignored (assumed to be 0). This option can be useful for experimenting with the effect of higher required coverage on hierarchical clustering without needing to recompute profiling each time.

    SNP frequency range to consider

    Except for positions satisfying one of the conditions below, SNP frequencies outside the specified range will be ignored (assumed to be 0). This option can be useful for focusing the clustering algorithm on a specific type of mutation.

    • All profiles as long as one profile meets constraint

      For any genomic position, if SNP frequency data from at least one profile at that position satisfied the frequency range constraint, then the SNP frequency data from all other profiles at that position will be considered (i.e. will not be assumed to be 0). This exception is enabled by default.

    • Position range +/- around position that meets constraint

      For any genomic position, if SNP frequency data from at least one profile at that position satisfied the frequency range constraint, then the SNP frequency that from all profiles for positions within a given range of that position will be considered (i.e. will not be assumed to be 0). This exception can be set only if the "All profiles..." exception above is enabled.

    Save generated shrunk sequences

    Setting this parameter to "yes" will generate and save a fasta file containing the sequences filtered according to the Advanced Parameters for SNP frequency range to consider.
